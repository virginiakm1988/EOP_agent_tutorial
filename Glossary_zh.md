# 術語表 — Agentic Engineering 速成課

本教程 Lab 1–6 中出現的術語，一句話定義。遇到不熟悉的名詞時可查閱此表。

| 術語 | 定義 |
|------|------|
| **API** | 應用程式介面：讓你的程式呼叫外部服務的方式（例如透過 OpenAI API 送出 prompt 並取得模型回覆）。 |
| **context window（脈絡視窗）** | 模型一次能「看到」的 token 數量上限；超過的輸入會被截斷或拒絕，只有視窗內的 token 會影響模型輸出。 |
| **conditional decoding（條件解碼）** | 根據模型在可能的下一個 token 上的機率分布，在*給定*目前 prompt 與先前脈絡的*條件下*，產生下一個 token（或工具選擇）的過程。 |
| **distribution over next tokens（下一個 token 的分布）** | 模型對每個可能的下一個 token（或工具名稱）所賦予的機率集合；工具選擇實質上就是從此分布取樣或取 argmax。 |
| **logits（對數機率／未歸一化分數）** | 模型對每個可能的下一個 token 給定的原始分數（尚未經 softmax）；logit 越高，歸一化後該 token 的機率越高。 |
| **LLM（大型語言模型）** | 一種根據前文預測下一個 token 的神經網路；用於對話、工具路由與結構化輸出。 |
| **prompt（提示）** | 你送給模型當作輸入的文字（與可選的結構化訊息）；它會制約模型的回覆，可包含系統指令、工具定義與使用者訊息。 |
| **schema（結構描述／綱要）** | 對允許的結構與型別所做的正式描述（例如工具參數的 JSON Schema）；用來約束模型或程式能產生與解析的內容。 |
| **state（agent state）（狀態／代理狀態）** | 代理在多輪對話或圖節點之間攜帶的結構化資料（訊息、工具結果、重試次數、草稿等），會隨工作流程執行而更新。 |
| **token** | 模型讀寫的最小文字單位（常對應一個詞或子詞）；prompt 與回覆的長度以 token 計。 |
| **tool call（工具呼叫）** | 模型發出的、要執行某個函式的結構化請求（例如 get_weather、search_docs），通常包含名稱與參數，由你的程式執行並回傳結果。 |
